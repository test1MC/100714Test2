Month End Procedures

 Documentation Published on

At the end of each month, several items should be taken care of in Dynamo to prepare for the next month of processing. Many of these should be run at the end of the year as well.

The applications on the Month End Procedures (M/E) menu are necessary for changing to a new month in Dynamo. However, before changing the booking month and updating, the operator should consider running some other applications that will ensure proper processing.

  • The Unvouchered Receipts Listing on the GLR menu should be run to ensure that all unvouchered receipts are linked to a purchase order and have the correct amount assigned to G/L accounts. This report will help ensure that all A/P invoices have been coded to properly reflect the PO#, Receipt#, and G/L account distribution.
  • The A/R Aging Report (ARR) should be run in order to forecast cash flow and keep an eye on collections.
  • The A/P Trial Balance Report (APR) should be run to view any variation between the General Ledger Accounts Payable Account and the Accounts Payable detail as of a specific date.
  • The Fixed Asset Depreciation Schedule (F/A) should be run since it applies depreciation to fixed assets and creates a G/L Posting.
  • The Salesperson Commission Report (M/E) should be run (typically by branch/sales managers and salespersons) to verify that commissions were calculated correctly.
  • Run Change Order Booking Month (M/E). This should be done as soon as the last order for the month being closed has been entered.
  • Run Month End Update (M/E).
  • G/L Period Ending Date Maintenance (GLU) - Open the new month and be sure to mark the prior month as closed once all transactions have been processed. This helps prevent accidental posting to the wrong G/L period.
  • Payments on Open Sales Orders (GLR) - If you take payments of any kind when entering a sales order, then this report will provide a check to ensure that all open orders with payments have been invoiced.
  • Daily Inventory Value (GLR) - This report compares the inventory values at the end of each day to the inventory value in the General Ledger. It is much easier to resolve a variance when discovered quickly.
  • Special Order Inventory Detail (GLR) - If you ever buy and sell items that are one-time sales and are not set up using Item Maintenance, this report can help determine if the sales order used the correct item# and cost for these special order items.
  • Inventory in Transit (GLR) - If you have multiple warehouse or store locations, this report can be used to verify that all completed transfer orders have been received and updated and that the dollar value of the inventory in transit agrees with the General Ledger.
  • Inventory Used in House (GLR) - If you take product out of inventory for your own use, you may need to pay State Use Tax if you did not pay State Sales Tax when these items were purchased. This report lists items taken out of your inventory for your own use for assistance in computing State Use Tax.
  • Physical Inventory (I/C) - Whether you take a cycle count every month or the entire store/warehouse at the end of the year, this application can make taking and checking your inventory a snap.
  • If you have a local server with a tape drive, then run the cleaning tape to ensure best performance of your tape drive before replacing with the next backup tape.

 

Some companies may wish to run some of the Sales Order Processing Reports (SOR menu) and/or Inventory Evaluation Reports (I/C menu) before updating to the new month.
