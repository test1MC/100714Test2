Export Data

 Last Revised: 07/07/11

This application is used to configure Dynamo data to be exported.  The data can be exported to tab delimited files or to a mysql database for use with the Excellware data mining/data warehousing application called DynaMine.  The data can be exported interactively using this menu application, or as an automated background process using Dynamo Tool CDS562.

Note:  This application requires that the DynaMine Database name, Login ID, and Password be specified using Company Information Maintenance (SMC) in order to export data into the DynaMine database.  These fields are not required when exporting to tab-delimited files.

Group The Export Data Group is a 4 character field used to define a group of data files that will be exported at the same time.
Description A brief description of the Export Data Group.

 

The Files & Fields view is used to describe what files and fields in each file, are a part of the Export Data Group.  A Group can include one or more files.

 

The Define DynaMine Tables option will create a DynaMine table for each file in the Group.  The table will be deleted if it already exists.  There will be no data in the table at the conclusion of this step.

The Export Data option is used to create the text file containing the exported data, or to export the data into the DynaMine database.

 

Some of the options shown have not yet been implemented.  Specifically the two options to export in XML format instead of csv format, and the option to export selected records instead of the all records, are incomplete.

DynaMine Setup

DynaMine is a data mining/data warehouse application based on Oracle's MySQL database, and other third party reporting tools such as those provided by Jaspersoft.  The steps to setup and configure DynaMine on a Linux server are described below.  The concepts would be similar for other operating systems.

DynaMine Setup & Configuration
Login as root, then run the commands show below
1 Install MySQL Check to see if the mysql packages are installed: rpm -q mysql mysql-server
If not installed, you can install on RHEL 5 or newer using: yum install mysql mysql-server

or on older versions: up2date mysql mysql-server
2 Configure MySQL daemon to run on reboot Check to see if myslqd is run upon reboot: chkconfig --list mysqld
Configure to run at run-levels 3, 4, and 5: chkconfig --level 345 mysqld on
3 Start mysqld Check to see if mysqld is running: service mysqld status
Start if required: service mysqld start
4 Create a database List existing databases:

mysql

show databases;

quit;

Add new database:

create database [db_name]; Recommended db_name is dynamoXX where XX is the Dynamo Company Code.

quit;
5 Create an operator List operators:

mysql

use mysql

select operator,host from operator;

quit;

Add operator:

mysql

use mysql

grant all on [db-name].* to [operatorname]@localhost identified by '[password]';

flush privileges;
quit;

For convenience, you can use dynamoXX for each of [db-name], [operatorname], and [password] fields if desired.
6 List tables mysql

use dynamoXX

show tables;

quit;

7 Configure DynaMine Use Company Information Maintenance (SMC) to specify the DynaMine database name, login ID, and password to use when exporting data.

There are several tools available that can browse and configure a MySQL database.  These can be valuable to verify tables are configured properly, data is exported, etc.  SQLyog Community Edition available for free download at http://www.webyog.com is an example of one very capable and useful product.
