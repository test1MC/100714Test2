Company Wide Parameters

Warehouse Information Maintenance

 Last Revised: 01/09/14

Use this program to edit information about warehouses entered in Dynamo.

Warehouse Information Maintenance Primary View
Warehouse Enter an existing warehouse code to maintain it or type a new character to create a new code for a warehouse.
Location Give a location of the warehouse; this is typically a city name.
Name Provide the name of the company.
Address City, State/Country, Zip Code, County Provide the address for this warehouse of the company. When you enter the zip code, the city, state, and county will populate.
Phone Provide the phone number for this warehouse of the company.
Fax Provide the fax number for this warehouse of the company.
G/L Location Specify which general ledger account funds for this warehouse are sent.
Type
S Standard Warehouse
D Drop Ship Warehouse
R Rental Only
V Vendor Returns
T Transfer Order in Transit
Active A warehouse can be active or inactive. An inactive warehouse could be no longer in use or could be not yet active.
Sequence Indicate the warehouse's priority over other warehouses (the lower the number the higher it shows up on a list of warehouse choices).
Supplying Warehouse Specify which warehouse supplies the one being maintained.
Location Mask Specify the location code format by using "A" to indicate letters and "N" to indicate numbers. For example, the code could be ANNN, so a location code might look like B123.
Physical Inventory Locations
1 Clear no locations Do not clear any locations from the computer's files after physical inventory.
2 Clear secondary locations not counted Clear only the secondary locations of an item (typically overstock locations) after physical inventory.
3 Clear all locations Clear all locations from the computer's files after physical inventory.
Release Orders to Dynascan If the operator has Dynascan, this feature can be turned on to release sales orders and transfer orders to warehouse once no more changes will be made. Dynascan will calculate the orders once the warehouse has received the information. Dynascan can be used to receive, put away, ship, or count orders.

Sales Order Processing Options
SOP Qty Shipped Default
Z Default Quantity Shipped to Zero
O Default Quantity Shipped to Quantity Ordered
A Default Quantity Shipped to Quantity Available
Packing List Quantities
B Leave Quantity Shipped & BO Blank
P Print Quantity Shipped & BO
Sort Packing List by Location Specify whether the packing list will be sorted by item location or not.
Auto Invoice when Shipment Processed Invoices can be completed automatically by Dynamo upon shipment or manually.

Transfer Order Processing Options
TOP Generation Sequence This option controls the sequence in which warehouses are processed. A lower numbered warehouse receives transfer orders before warehouses with higher numbers. If "0" is specified, the warehouse is excluded.
TOP Qty Shipped Default

Choose one of the following options the quantity shipped will default to for transfer orders for this warehouse.

Z Default Quantity Shipped to Zero
O Default Quantity Shipped to Quantity Ordered
A Default Quantity Shipped to Quantity Available
Transfer Orders Enter how the transfer orders will be printed/faxed/emailed by selecting a default printer.
