Company Information Maintenance

 Last Revised: 04/21/16

This program is used to maintain company specific data.  The company name, address, and phone numbers are printed on certain Dynamo documents and can be modified as required.  operators should log out and back in to obtain any changes made using Company Information Maintenance.

Fax# Primary company fax# used on certain Dynamo documents when not using the branch or warehouse fax#.

Contact Info Called Rtn

This routine is used in conjunction with the Dynamo Tools printer selection routine, message and memo sending programs, to enable selection of a customer, vendor, salesman, name, address, email address, phone numbers, etc. without having to enter this information manually nor use copy and paste. The routine is called when the operator touches F2 at the Contact Name entry.

In most cases this routine name should be left at the default value of SW110. If, however, your system uses Dynamo Tools but not Dynamo, then you may want to create a Contact Info Called Routine to work with your application's customer, vendor, salesman, etc., files. For more information, refer to the Dynamo Tools documentation for SW110.
Log System Activity Set to Y to enable logging of each menu selection made for printing the System Activity Log (SMC). If enabled, be sure to print or purge the log periodically as it can become quite long.

Licensed DT operators Only used when multiple companies are running on the same server, and there is a need to limit the number of Dynamo licenses by this company that can be used at any time.

Prefix

 

 

 

BBj Prefix Programs can be stored in 3 formats, bbx tokenized, text only, and BBj tokenized. When running BBj applications, if the program is not in BBj tokenized format, then BBj will convert the program to BBj tokenized prior to execution. Although BBj uses a program cache where converted programs are retained for future use, you can eliminate the time required for program conversion by saving programs in BBj tokenized format in a special directory. For example, Dynamo stores programs in bbx format in the DY/ folder, and the same program is saved in BBj format in the DY/BBj/ folder. If an entry is made in this BBj Prefix field, and you are running BBj, either by menu, stored procedure, trigger, etc., then Dynamo Tools will include this BBj Prefix entry in the front of the prefix so that BBj will find the BBj tokenized version of the program first, eliminating the conversion time.
Prefix

The company specific prefix is used as a search path for programs and data files. It does not need to include company CD/, nor the current company XX/, nor any folders that contain templates. In most cases, each entry in the prefix should end with a forward slash (/). The order that the entries appear in the list is the order that bbx will search for files and programs. The company specific prefix is typically not changed by end operators.

The prefix consists of the following segments:

  1. Any prefix entry as entered in the Dynamo Tools menu using the "P" (Prefix) or "J" (Job/Project) menu option.
  2. If the Dynamo Tools install directory is across a data server, then the Dynamo Tools install directory (SMSDIR) is included next in the prefix.
  3. When using another menu application and not directly logging in to the Dynamo Tools menu, the prefix at the time CDS000 is called by the other menu application is included next in the prefix.
  4. If a system specific folder based on the Excellware License Control#, "L"+STR(LCN:"00000") exists, then this is included next in the prefix.
  5. The active company code XX/ is next.  This entry will include the data server prefix when the Dynamo Tools install directory is across the data server.
  6. If running BBj, then CD/BBj/ is next (plus data server prefix when applicable)
  7. If the active company code is not CD, then CD/ is next (plus data server prefix when applicable)
  8. Any prefix specified using Menu Maintenance for the active menu is next.
  9. If the BBj Prefix described above is non-blank, it is next.
  10. Finally the company specific Prefix specified here is next.

Note that after the prefix is built, any parenthetic reference, i.e., (MYGLOBAL) is replaced with the value of the global variable name specified between the parentheses.

Note that the above prefix sequence is applicable when running the Dynamo Tools menu. Applications that do not use the Dynamo Tools menu, but use CDS011, including background tasks, Dynaweb, BBj stored procedures, triggers, etc. only use prefix segment numbers 5, 6, 7, 9, and 10 as listed above.

Development Projects

 

Dynamo Tools provides a mechanism to manage and install programs and other file types. New projects are defined using Project Maintenance (FMS). Projects can be activated, i.e., by moving program files and other files into the Default Directories that are maintained here. The activation process can be completed immediately after importing a project, or at a later time using Project Maintenance.

The Next Job# field is updated automatically as jobs are created using Project Maintenance (FMS), but can also be set manually to specify the starting Job#.

Dynafax

Dynafax is an option to send faxes without a fax modem, special fax software, or a telephone line.  Dynafax delivers the fax using internet email to a third party, who transmits the document to the recipient's fax machine. Dynamo Tools supports both fax87.com and Esker as the service provider.

Dynafax Default Notification Only applies when using Dynafax via Esker on Demand. Sets the Company default value used to notify the sender that a Dynafax was delivered or not. Can be overridden by operator on the Printer Options View when sending a Dynafax.
FEmail sender only if fax fails
AEmail sender always (whether fax succeeds or fails)
SEmail sender only if fax succeeds

Dynamine

Dynamine is a Data Mining/Data Warehousing application that can be used for historical reporting of Dynamo transactions.

Dynamine View
Database Name Specify the Dynamine database name used with the Export Data (FMS) application.  The suggested database name is dynamo followed by the two character company code.
Dynamine Login ID The Login ID used for the Dynamine database.  This login is used by the Export Data (FMS) application to populate the Dynamine database.
Dynamine Password The password associated with the Dynamine Login ID
Dynastore

Dynastore is a report archiving server where documents printed using PDF, or files created in spreadsheet format by printing, are archived in the "cloud."  Reports can be retrieved or deleted using the Dynastore Reports option on the SMS menu.

Dynastore Option This parameter sets the default Dynastore option when printing.
NDo not archive report
SArchive with Standard Access, meaning that anyone with security permissions to run the report, can also view archived reports
PArchive with Personal Access only, meaning that only the operator that printed the report can view the archived copy.
AArchive with no access restrictions.  Anyone may view the archived report.
Dynastore ID This ID uniquely identifies this Dynamo company.  Contact Excellware to obtain a Dynastore ID and Key.
Dynastore Key This is like a password for the Dynastore ID.
