Form Processing

 Last Revised: 03/22/11

This application is used to produce printed forms.  The form can contain text, dates, check boxes, and radio buttons.  The empty form is maintained using a word processor.  A called routine is provided to return an empty templated data structure used to populate each of the fields on the form.  Another called routine is used to merge the variable data with the empty form, and to send the completed form to a PCL compatible printer or PDF file.

 

Heading
Form ID A 4 character identifier for the form.  Use a valid string variable name, i.e., begins with a letter, contains only letters, numbers, and underscore.
Description A brief description of the form
Orientation
P Portrait
L Landscape
Overlay File Name The empty form can be maintained using a Word Processor.  Email the document to Excellware Support Manager, who will process the document and return to you an overlay file.  Specify the path to the overlay file name here.
Use Bold Text
N The merged data is printed using the normal font weight
Y The merged data is printed using bold font weight

 

Form Fields
Field Name Each field on the form must be assigned a unique bbx field name.  Normal field naming rules apply, i.e., starts with a letter, only contains letters, numbers, and underscores.
Description Optional additional descriptive information about the field.
Type
T Text - text field consisting of one or more rows
D Date
R Radio Button - only one radio button per variable can be selected at a time, for example, a Yes/No field is a radio button, as the value can be Y or N, but not both.
C Checkbox - one or more boxes may be checked.
Size The use of the Size field varies with the field Type.
TypeSize
TMaximum number of characters in field
DAlways 8
RAlways 1
CAlways 1
Repeat The use of the Repeat field also varies with the field Type.
TypeRepeat
TThe number of rows in the text field
DThe number of different dates in the date field
RThe number of different options in the radio button field
CThe number of different checkboxes in the checkbox field
Row The dot row of the baseline where the variable field is placed on the form.  There are 300 dot rows per inch.  The Row can best be estimated using 'Print Sample Form', 'Print Grid over Form' option.  This will print a grid of numbered row and column lines every 50 dots.  Once the Row is estimated, you can fine tune the position by printing the sample form.
Col The dot column of the left side of the position where the variable field is placed on the form.

 

Repeating Fields
Description Optional additional description of the element within the repeating field
Value The single character value associated with the element.  The Value must be different for each element in the repeating field
Row Dot Row for placement of repeating element on page
Col Dot Column for placement of repeating element on page
Print Sample Form Options
Print Grid over Form Use this option to assist in estimating the Row and Col that each form element should appear on the page.
Print All Fields This option will print text in each position of the form to assist in fine tuning the best Row and Col.
Print Typical Form This option is similar to the 'Print All Fields' option, but only prints one of the Radio Button elements, as would be typical of the actual form.
Called Routines used by developer
CDS604 Return Template for Form
CDS606 Merge Form with Variable Data & Print
