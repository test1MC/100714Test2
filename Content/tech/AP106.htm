AP106

A/P Compute invoiced amounts for PO Receipt

 Last Revised: 04/18/08

This routine is used to determine invoiced amounts for a specific receipt.  It also is used to determine the date that a receipt can be closed, i.e., the G/L posting date of the vendor invoice that completed charges for the receipt.

It is called during A/P Invoice Data Entry to enable the operator to select the receipt or receipts that apply to the invoice.  It is also called during A/P Invoice update to set the date that the receipt was closed, and by the Unvouchered Receipts Listing.

CALL "AP106", Y$, REC$, AP106$, { UVRDATE }

AP106 Arguments
Field Passed To/From Description
Y$ To Session Control Variable
REC$ To Passed as PO50$, or PO52$, or AP15$
AP106.RECEIVED From Received Value
AP106.THISINVOICE From If REC$ is passed as AP15$, then AP106.THISINVOICE will be the amount linked to this vendor invoice.
AP106.OTHINVOICE From If REC$ is passed as AP15$, then AP106.OTHINVOICE will be the amount linked to any other vendor invoice.
AP106.BILLED From Total amount billed by vendor.
AP106.BALANCE From Amount left to be billed, or over billed (if negative).
AP106.CLOSED From G/L Posting date of the vendor invoice that resulted in the receipt being billed completely.  Used to set PO50.CLOSED which is used by the Unvouchered Receipts Listing.
UVRDATE To Optional Unvouchered Receipts Date.  If passed, then transactions with a G/L Posting date after UVRDATE will be excluded.  This is used when printing the Unvouchered Receipts Listing as of an operator entered date.
