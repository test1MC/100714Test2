IM600

Compute Cost for PO

 Last Revised: 08/03/07

This routine is used by Purchase Order Processing applications to obtain the default unit cost and related fields for an item.

Additional description.

CALL "IM600", Y$, IM01$, VENDNO$, DATE, QTY, IM600$

IM600 Arguments
Field Passed To/From Description
Y$ To Session Control Variable
IM01$ To Item Record from Item File.
VENDNO$ To Optional.  Needed by IM600 routine if cost varies by vendor.
DATE To Optional.  For date sensitive costs, this date would be passed to IM600 as the order date, so that it could compute the vendor cost as of that date.
QTY To Optional.  Needed by IM600 routine if cost varies by quantity ordered.
IM600$ From
COST Unit Cost
CU Costing Unit, blank, C=per hundred, M=per thousand, etc.
DISCPCT If COST is further discounted by a percentage, the discount percentage would be returned here.
FETCOST Federal Excise Tax cost when applicable.
CORECOST Core cost when applicable.
