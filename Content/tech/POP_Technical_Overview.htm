Dynamo purchase order processing

Technical Overview

 Last Revised: 01/15/09

Dynamo Purchase Order Processing provides the ability to enter, maintain, and send purchase orders to vendors, and to receive materials ordered into inventory.

There are many small programs involved this application.  Refer to the Flow Diagrams below for a visual representation of the programs.

Purchase Order Data Files

Name Description Usage
XXPO00 Purchase Order Parameters Contains next purchase order# to be used and other parameters that control Purchase Order Processing.  Single record retrieved using SW005.
XXPO40 PO Heading Record for each purchase order
XXPO41 PO Detail Record for each purchase order line item
XXPO42 PO Timestamps Record for each purchase order timestamp
XXPO43 PO Notepad Text This file contains all notepad text associated with the PO.
First key segmentNotepad contents
PO number plus "INTERNAL"Internal notepad not printed on purchase order
PO number plus "EXTERNAL"Notepad printed at start of PO before first line item
PO number plus HTA(PO41.PO41ID)Notepad associated with line item, printed on purchase order below line item.
STANDARD_NOTEPADCompany Wide Notepad that prints on all purchase orders after all line items.
XXPO50 PO Receipts Heading Record for each receipt.  A receipt is defined as all items that appear on the same vendor packing slip.  A receipt can include items from more than one purchase order.
XXPO51 PO Receipts Detail Record for each item received per PO Line Item.  If the receipt includes multiple PO's, and a particular item is included on more than one PO, then there will be multiple PO51 records for this item.
XXPO52 PO Receipts by PO# Record for each PO# included on a receipt.  Typically a receipt includes items from a single PO, but some vendors consolidate all open orders, especially backorders, and ship them together in a single shipment on the same packing slip.
XXPO53 PO Receipts Timestamp Record for each PO Receipt timestamp
XXPO54 PO Receipts by Skid# Record for each item received by skid#.  Used when it is necessary to produce a skid manifest listing items received on each skid.  An item may be located on multiple skids.

CDS037/38 Type External Routines

Program Input Output
PO131 PO41$
QTYRCVD Total Qty Received for this line item
BALANCE Qty open for this line item
TOTAL Total Purchase Cost for line item (sum of Product Cost, FET Cost, and Core Costs)
PO132 PO41$
QTYRCVD Total Qty Received for this line item
QTYOPEN Qty open for this line item
DATERCVD$ Text version of Date Received.  If only one receipt for this line item, then DATERCVD$ will contain the date in MM/DD/YY format.  If multiple receipts then DATERCVD$ will contain "Multiple".
VALRCVD Value received including product cost and FET.
VALOPEN Value open including product cost and FET.
PO133 PO51$
VALUE The Value of the product received for one line item, including product cost and FET, but excluding Core Value
PO134 PO50$
VALUE The Value of the product received for all line items on a receipt, including product cost and FET, but excluding Core Value
PO135 PO41$
DATEDUE The date due for the line item if specified, otherwise the date due for the PO
PO136 PO50$
PONOS$ A list of purchase order numbers included on the receipt abbreviated to 21 characters.  This is a display/print field with leading spaces removed.
PO137 PO40$
VALRCVD The value of the product received for all line items on the PO, including product cost and FET, but excluding Core Value
VALOPEN The value of the product remaining open for all line items on the PO.
