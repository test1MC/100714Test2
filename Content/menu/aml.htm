Assortment Maintenance and Listings

AML Menu Overview

 Last Revised: 02/19/15

An Assortment, sometimes called an Assembly, is a collection of multiple products that can be sold as a single item.  The applications on this menu are used to define what items and quantity are included in an assortment.  Each component of an assortment, as well as the 'parent' assortment, require Dynamo Item numbers.

Glossary

Term Meaning and usage
Assortment

A group of related products or services keyed to a single Parent Item.
All elements in an Assortment can be sold by entering the Parent Item on the order.
Parent Item A valid Item Number in the company's Item Master File.
It will be used only to define the Assortment and cannot exist on its own.

A Parent Item can NOT be a component of another Assortment.
Component Item A valid Item Number in the company's Item Master File, or in some cases,
a non-stocked item.
It is one of the elements that make up the Assortment.

It can NOT be the Parent Item of another Assortment.

Assortment Type

 The Type Code controls the way an assortment behaves in the system. 
Standard Code = Blank.
Parent Item must be built using inventory transactions before it can be sold.
Sales are tracked for Parent Item only.
'T' Code = T.
Parent Item can be built using inventory transactions before it is sold, or it will be built from available components automatically when the order is entered.
Sales are tracked for the Parent Item only
'X' Code = X.
Same behavior as Type='T', except that sales are tracked for Component Items as well as for the Parent Item.
