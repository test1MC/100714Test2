Create Rush Backorder

 Last Revised: 07/12/13

With this application, the operator can create a rush backorder for a customer. When an Invoice Update is run, backorders are created. However, some operators only run Invoice Updates the following morning, which can cause issues for time-sensitive customer orders. Create Rush Backorder can be run to bypass the need for an Invoice Update. The back order will then be created immediately.

Note: When creating a rush back order, it is important NOT to modify the original order before the Invoice Update.
