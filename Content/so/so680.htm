Sales Order Processing Reports

Sales Transaction Listing

 Last Revised: 04/29/14

Sales Transaction Listing is used to view and print a listing of all sales transactions in Dynamo. It will contain the branch, the warehouse, the invoice date, the sales order number, the order date, the name, the product category, the tax, the item & its description, the quantity, the amount, and the cost of each sales transaction.

Operator-Specified Fields
Starting Date Specify the beginning date to be included on the listing.
Ending Date Specify the ending date to be included on the listing.
Branch Choose the desired branch.
Product Category Types

Enable the following to display them on the listing:

  • costed
  • uncosted
  • other
Product Class Type "Y" to select a specific product class or "N" to include all of them.
Item

Choose one of the following:

  • All Items
  • All Items on Item File
  • All Items not on Item File
  • Operator Specified Item
  • Operated Specified Starting Characters of Item#

 

If "All Items" is selected, the report may take a few minutes to generate depending on how many items are in the system. If the operator prefers to see transactions for one item, "Operator Specified Item" should be chosen.

Customer Type "Y" to select a specific customer or "N" to include them all. If all customers will be on the report, it may take a few minutes to generate.
Customer Type Type "Y" to select a specific customer type or "N" to include them all.
Sale/Rental/Used Rental Enable this options for the display of sale, rental, and used items on the listing.
Extended Amount Range Indicate the desired range of extended amount (e.g., only items $500 and higher). The extended amount is the quantity times the price of the item.
Sequence

Choose one of the following information selections:

  • Branch+Warehouse+Invoice+SalesOrder#
  • Product Class+InvoiceDate+SalesOrder#
  • Product Class+Margin%
  • CustType+Customer+InvoiceDate+SalesOrder#
  • Item#+InvoiceDate+SalesOrder#
