System Management Configuration

Install/Update Software Packages

 Last Revised: 05/11/09

This program is used to update Dynamo System Management or Application Software on Unix servers.

Use the "V" option at any Dynamo menu to display the version of software packages currently installed.

The following packages are distributed by Excellware.  Other software providers may install their packages using this method.

Package Code Name Description
SMS System Management Software The run-time component of Dynamo Tools, required on all servers
FMS File Management Software The portion of Dynamo Tools used by Software Developers that includes Data Dictionary Maintenance that is not required to be installed on each server.  It is often desirable not to install the FMS package on the live server when there is a separate development server used for software development and testing.  This will prevent changes to the Data Dictionary on the live server so they will not be over-written when exporting Data Dictionary data from the development server to the live server.
WMS Dynaweb Management Software The portion of Dynamo Tools required for Dynaweb applications.  Dynaweb provides a browser based interface to Dynamo applications and data.
ZIP Zip Code Database Includes the programs and data files that provides all valid 5 digit US Zip codes along with the city and state code.  The programs and files included in this optional package are used when entering addresses.  The operator can enter a US zip code, then select from the list of valid cities.
DAP Dynamo Accounts Payable Programs and data files required to run the Dynamo Accounts Payable application
DGL Dynamo General Ledger Programs and data files required to run the Dynamo General Ledger application

This program requires all other operators to be logged off and requires the System Administrator (root) password.

Prior to executing this application, download the software image file as directed by Excellware or your software provider.  Save this file in the $SMSDIR/tmp folder on the server.

The operator will be shown a list of the software packages to be updated before the update takes place.

Technical Notes

The installation procedure involves calling a pre-installation routine, installing, (i.e., copying the files of the package into the desired directories), then calling a post-installation routine.  In order to use the same media for both BBx Progression 3, 4, and 5 target systems, the pre and post installation programs for P3 and P4+ are named differently.

The installation routines are called programs with six character program names. The first three characters of the program are the same as the package. The last three characters of the program name can be determined using the table below.  The installation routines must be placed in the $SMSDIR/pkg/ directory prior to building the software package.

Last characters of conversion program   BBx Progression Level at update    Pre/Post installation    
000 P3 Pre
001 P3 Post
002 P4 or higher Pre
003 P4 or higher Post
005 P4 or higher Executed as non-root operator upon next login.

Note that the programs PKG000 and PKG001 must be P3 programs.

The ENTER list for these installation routines includes:

CDS630 Enter List
Y$ standard terminal control variable
OLDVER$ a template based string describing the previous version of the package. The template fields available include: 
PKGthree character package code
NAMpackage description
REVexisting revision
RELDATrelease date in julian format
PKGREC$ a template based string describing the package being installed. It has the same template fields available as in OLDVER$
PKGS$ this string variable describes information about all packages included on the media.  PKGS$ includes 80 characters for each package.  The structure of each 80 character slot is the same as in OLDVER$.  However, there is no template associated with PKGS$, but you can associate the same template used with PKGREC$ with any 80 characters in PKGS$.
DIR$ the directory where the tar based package files are located. Currently DIR$ is set to tmpins/.

The conversion routines may signal CDS630 to abort the installation.  This may be required when the conversion routine cannot locate necessary files or determines there is insufficient disc space, etc.  To signal CDS630 to abort the installation of this package and not to install any other packages on the media, exit with an error value of 99.  This is accomplished using a statement EXIT 99 as opposed to simply EXIT.

In some cases, you may want to bypass the installation of a package but continue installing other packages on the media.  This might be done when the media contains many packages, but the installation routine determines which packages are to be installed.  In that case use an EXIT 98.  The version record will not be written or replaced.
