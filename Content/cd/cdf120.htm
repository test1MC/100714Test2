File Management Software

Documentation & Help Text Maintenance

 Last Revised: 01/10/14

This application is used to define and maintain documentation and help text documents.  It supports both web based and traditional Notepad based documents.  It is designed to work with the FacetWin terminal emulator to launch Microsoft Front Page as the html editor.

Refer to each section for additional information.

Web Based - used to define, edit, or delete a web based (html) document
Web Control - used to specify folders and template for web based documents
Notepad - used to maintain notepad help text documents

Web Based

Web Based documentation and Help Text is preferred over the traditional Notepad Help Text because web based documents can have various fonts and colors, logos, graphics, tables, etc.

Web Based documentation, however, requires a web server such as Apache running on the server to serve up the web pages as requested by operators.

The first entry is to select the Document Type from those defined in Web Control described below.

 

Once the document type is determined, the remaining entries are determined by the Placeholders used for that document type.

Placeholder  
[MENU] Operator will be prompted to enter or select from a list any of the three letter Dynamo Menu Codes that exist for the current company.
[APPID] Operator will be prompted to specify whether the document is for a Menu Selection or the Menu Overview document.  For Menu Selections, the operator is prompted to select one of the applications attached to that Menu.
[DOCNAME] Should this Placeholder be contained within the document file name, then the operator will be prompted to enter the document name or select an existing document from the published or unpublished folder (when applicable).
[PROGNAME] Should this Placeholder be contained within the document file name, then the operator will be prompted to enter the program name.  Typically used when documenting called programs.
[JOBNO] Operator will be prompted to enter a job# or select a job from a list for a selected customer.
[CUSTALPHA] Operator will be prompted to enter a customer# or select a customer by searching by Customer Alpha Sort Name, Contact Name, Phone#, Zip Code, etc.

Once the document file name has been determined, the operator can edit or delete the document.

If the document does not exist and there is a template document specified in Web Control, then the template document is used to create the new document.  Any Placeholders contained within the template document are replaced with their values.  See table of Placeholder fields below.

When using the FacetWin terminal emulator, the edit option will launch Microsoft FrontPage to edit the document.  Since this application is designed to be run from FacetWin with path to documents from the Unix/Linux perspective, there is a convention used so that each Unix/Linux path can be associated with the Windows Path to the same file.  The convention is that the first folder of the Unix/Linux path is ignored. See examples below. The actual server name is used in the Windows Path.

If using FacetWin to enable Windows to access shares on the Unix/Linux server, then a Share needs to be defined in the [facetwin install directory]/Share file to enable access from Windows applications such as Microsoft Front Page.

Web Control

This option is used to specify information specific to each type of document.

Type A single character used to identify the type of document
Description The description for the type of document, such as 'Application,' 'Called Routine,' 'Other Document,' etc.
Path Published The path to the document file using the Unix path.  Note that the Path Published and Unpublished typically contain one or more Placeholders to determine the document path or file name.  The Placeholders are indicated by use of square brackets.  See below for example and list of all valid Placeholders.
Path Unpublished In some cases, operators that manage documents place their original document in the 'Unpublished' folder.  Once the document is approved for publishing, it is typically moved to the published folder.  If you do not require a review process, then this field can be omitted.
Template File The full path to document that is used as a template to create new documents.  The Template File is optional.  Note that the Template File can contain placeholders for text replaced when creating a new document.  See list of supported placeholders below. Using a Template file can reduce new document creation time, and insure that all operators creating documents use the same style sheet, same heading format, etc.  There is also an option to 'Edit template using Front Page.'

The template file can contain Placeholder text contained within square brackets [ ].  The Placeholders supported are described below.  When creating a new document, the template file is used, and all placeholders contained within the template are replaced by their values.  Then the new document is saved in the appropriate folder.

General Purpose Placeholders
[MENU] For applications, the three letter Dynamo Menu Code where the application is located.  Note that any slash "/" character in the Menu Code is replaced with an underscore "_".
[DOCNAME] The full document name
[PROGNAME] The program name as specified in the Dynamo Menu
[APPID] The application ID, which is the program/file name specified in Dynamo Tools Menu Maintenance.  For Menu Overview documentation, the APPID is Dynamo Menu Code converted to lower case.  Any slash (/) in the Menu Code is converted to an underscore (_), i.e., Menu Code A/P becomes a_p.
[TITLE] For programs, the title is the Program Name followed by a hyphen then the program description S187.DESC$ as returned by CDS187.  For example, the TITLE for CDS069 is "CDS069 - Input Routine".
For jobs, the title is the first line of the job description.
For menu overview documents, the title is the Menu Code followed by the Menu Description, i.e., SMS System Management Software.
For menu applications, the title is the application description from the menu, i.e., File Utility.
[DESC] For programs, the description is S187.DESC$ as returned by CDS187.
For jobs, the description is the complete job description with each line of the description on a separate display line.
For menu overview documents, the description is the Menu Description, i.e., System Management Software.
For menu applications, the description is the application description from the menu, i.e., File Utility.
[MENUTITLE] For menu applications, MENUTITLE is the Menu Description of the menu where the application is attached, i.e., File Management Software.
Program Placeholders
[REVLINK] Currently applies to Dynamo and Dynamo Tool programs only, this placeholder is used to create the hyperlink for the program revision date which displays program revision notes.
[REVLINKEND] Used with [REVLINK] to indicate the place for the </a> hyperlink tag.
[ARGUMENTS] The list of variables to be passed to a called program.  The values are derived from the Called Program Data Dictionary, when the
[TABARGS] The table of variable names and usage to be passed to a called program.  This placeholder includes the table row and table cell html tags as required.
operator Placeholders - information about the operator currently logged in as specified in Operator Information Maintenance
[LOGINID] The current operator's login ID (from Y.OPERID$)
[OPERNAME] The name of the operator
[EMAIL] The email address of the operator
Date & Time Placeholders
[DATE] The current date in MM/DD/YY format
[LONGDATE] The current date in Sun January 1, 2009 format
Job Tracking Placeholders - The following placeholders apply only when using the Dynamo Tools Job Tracking Software.
[JOBNO] The job#
[CUSTNO] The customer#
[CUSTALPHA] The lower case format of the customer alpha sort
[CUSTNAME] The customer name
[COST] The cost of the job in text form such as "No Charge", "Actual Time", "Not to exceed $xxx", "Fixed amount of $xxx"

Notepad

Notepad help text is stored in file XXSK57.  The Help Text ID can be up to 10 characters long.  The Help Text is linked to a particular operator data entry using the PROMPT$ variable in CDS039/CDS069.  At the end of the PROMPT$ variable, include "{XXSK57,HELPTEXTID}".  This will cause the prompt to include the phrase: F3=Help.  If the operator touches the F3 key, the option for 'Item Level Help' will appear, and if selected, will display the Help Text defined in file XXSK57 with Help Text ID of 'HELPTEXTID'.

Example: This example from the Program Utility - Search option is used to provide help text for the Search Argument Entry
In this example, the Help Text is stored in file CDSK57 with a Help Text ID of 'CDS150-SA'.
1260 CALL "CDS039",1,74,4,L,"+search argument+{CDSK57,CDS150-SA}","A",ARGS$,"SEARCHARG"
