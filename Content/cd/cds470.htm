System Management Software

Operator Information Maintenance

 Last Revised: 11/14/14

This application is used by Security Administrators to:

  • Create new Operators
  • Set/Change Operator Profile, Preferences, & Security Options
  • Define/Remove Operating System Login ID, password, and other operating system options
Profile

The Profile view is used to specify a unique Operator ID, the user's Name, Department, Phone numbers, etc.

Profile
Operator ID The Operator ID is a 10 character field that is used to login to Dynamo. Some older versions of AIX and SCO operating systems have an 8 character maximum. The Operator ID is typically all lower case, should not contain internal spaces or other special character other than a period and the underscore. Typical operator ID's are the user's initials, first letter of first name followed by up to 9 characters in last name, first name plus first character of last name.
Name Operator Name
Operator's Initials Used in some places where there is less space available to display the name
Department/Location Used to specify the operator's title, department, or work location
Phone# Four telephone numbers can be stored including work, home, cell, etc.
Extension Primary Telephone Number Extension if applicable
Email Address This email address is used as the sending email address for any emails sent from Dynamo by this operator.  If not specified the email will be sent from customer@excellware.com.
Windows Login If using a Microsoft Windows PC, specify the Windows Login Name here.  It must match exactly the Windows Login Name as specified in Windows Control Panel, operator Accounts.  It is required for an operator to access shared files located on the server from Windows applications such as Microsoft Office, and Windows Explorer.  Best practice is not to have any spaces in the Windows Login.  If spaces are desired, then include quotes around the Login.  It is possible to have multiple Windows Logins separated by a blank space as shown in the example above.
Dynamo Password The Dynamo Password is used with Dynastart.  You will be prompted to enter the password twice.  No operating system password is required when using Dynastart.  Dynastart can initially be launched Dynamo from a web browser, and can be launched from a shortcut located on the desktop, Applications folder, or other location after initial launch from the browser.
Security Group

In most organizations, there are groups of users that all have the same security settings. By first creating a role user (Security Group) such as "_counter" and defining the required security settings, you can then assign all operators that share that same security to this Security Group.

If this Operator requires their own security settings, then leave this Security Group field blank.

If this Operator can use the security settings of the role based Security Group, then specify the Security Group that this operator should use.

Preferences

The Preferences View can be used to enter any special preferences for the user. Note that the user can edit the fields on this View using My Profile and Preferences on the SMS menu.

Preferences
Font The display font used with Dynastart can be set here. If no font is set, or the font does not exist on the workstation, then a default font will be used
Font Size The size of the font can be specified when using Dynastart. A default size will be used if not specified.
Company Code at Login On a system with multiple companies, you can specify a company code here which will be used when Dynamo is started for this operator.  If not specified Dynamo will be started in the last company accessed by the operator.
Menu Code at Login Dynamo will be started at this menu code.  If not specified, then menu code SYS will be used if present and if the operator has access to the SYS menu.  Otherwise Dynamo will start in the first menu the operator is authorized to access.
Function Keys Specify a Function Key Group Code to override the default Function Key Group Code of "STD."  Function Key Groups are maintained using Terminal Function Key Maintenance on the SMC menu.
Colors Dynamo screen colors are set by company using Company Information Maintenance (SMC).  However, operators that prefer other colors can specify them here.  For systems with multiple company codes, you may specify different colors for each company.  You may also define colors for company ** which will apply to all company codes.  There are four sets of colors that can be selected:
  • Main window dim and bold text and background colors
  • Hot-key (F1) text and background colors for the window border, option, and highlighted option
  • Option (F2) text and background colors for the window border, option, and highlighted option
  • Help (F3) text and background colors for the window border, option, and highlighted option

The initial display indicates which companies have colors specified.
Source Code Editor Path Software Developers can specify the editor to use when editing programs using Dynastart. This path will be the path on the local workstation.
Security Information

This view is used to set basic security options for the user.

 

Login Enabled This setting can be used to prevent an operator from logging in without permanently deleting the user.
Maximum # of Hoteys A Hotkey is used to launch another instance of Dynamo while suspending the current session. A user returns to the prior session when they close or leave the hotkey session. In order to prevent a user from getting confused with too many layered sessions created using a Hotkey, you can limit the maximum Hotkey depth with this setting. A Maximum Hotkey of 2 is typically sufficient for most users.
Message Security

Dynamo Tools has a messaging system where users can send message to one or more other users.

1 Only Read their own messages This setting is satisfactory for most users where they can only read messages sent to their operator ID.
2 Read all non-private messages This setting will allow users to read messages that are not marked as private, that were sent to some other user.
3 Read all messages This setting allows the user to read all messages, including those marked as private.
Broadcast Messages

There is an option to Broadcast a message to all users.

Y Broadcast message to all users includes this user This would be the typical setting.
N Broadcast message to all users excludes this user This setting would be used for users that do not login.
S Exclude this user, but include all 'Same as' users This setting would be used for role based Security Group users where a user does not login, but there are other users that share this same Security Group that should receive any broadcasted message.
Security Administrator Only users that are authorized to run this Operator Information Maintenance application should be defined as a Security Administrator. You can have more than one Security Administrator, but most users should not have this special designation.
Restricted File Utility Functions This setting can be used to restrict a user from certain risky File Utility (SMU) operations such as initializing or erasing, or renaming a data file, or using the General File Maintenance application. A "Y" setting means that this user is permitted to use the Restricted File Utility Options. Most users should not have this option enabled.
Console Mode The setting is used to permit a user to enter BBx/BBj console mode and enter business basic commands. It should be disabled for most users.
Set Job Prefix A Job Prefix can be set temporarily to test an application before activating a software project for all users. Only users that might need to perform such pre-activation testing should have this option enabled.
Early License Renewal Message Dynamo Tools are a licensed product that requires an annual license registration. This setting is used to indicate the operators who should be notified before others, that the license is getting close to renewal time. License renewal is accomplished via a brief telephone call to Excellware where license control number and activation keys are entered.
Bypass Company License Check Dynamo is licensed based on the maximum number of simultaneous users. Only Excellware staff members should have this parameter set to Y so their access does not count toward the number of currently licensed users.
Enable Double Vision Double Vision is a remote control application that enables a user to view another user's screen. This setting should only be set to Y for users that are authorized to view another's screen.
Kill User Process Typically a user will log out of Dynamo when they are finished for the day. However, sometimes the session disconnects from the server, or the application is stuck in some kind of infinite loop, and needs to be killed. This setting should only be set to Y for users that are authorized to kill (abort) another user's running session.
Remote Location Access This setting enables the user to connect to a remote location such as a branch office, to remotely control a workstation, configure a printer, etc.
Delete Installed Software Package Software packages installed can be viewed using the V menu option. This setting enables the user to delete a software package from the list.
Printer Menu Options

This view is used to enable/disable the user from printing to special output device types.

Fax This setting is used to allow the user to print reports to a fax machine. Faxing at the system level must be enabled and configured for either Dynafax or VSI*FAX using Dynamo Configuration Maintenance (SMC).
Email This setting is used to allow a user to email reports.  Emailing must be configured on the linux/unix server.
File This setting is used to allow a user to print a report creating a text file, typically tab delimited for opening by a spread sheet or text editing application.
PDF This setting is user to create reports in Adobe PDF format. It requires the third party PDF creation tools installed on the Linux server.
Company and Menu Access

This view is used to maintain security access by company and menu code. Some systems have multiple Dynamo companies installed on the same server. Use this view to limit which companies a user may access, and what menus, and security levels they should have for each company.

In this example below, user willham has access only to company code UG, with access to the menu codes and security levels shown. The user would not have access to any other menu code since the wild card menu code of *** is not listed.

The Dynamo Security system will allow a user to run an application on a menu if:

  1. the user is authorized to access the current company code
  2. the user's security level for the menu code is greater than or equal to the security level for the application.

This program is used to assign the security level to the operator. Menu Control Maintenance (SMC) is used to assign the security level to each menu, and each application on the menu.

Operating System Settings

This view is used to define/remove the operator's operating system login ID and related information. This view is not applicable on Windows systems. A smaller list of options exist on non-linux systems.

Login Exists Dynastart users do not require an Operating System Login ID. But users that run a terminal emulator such as FacetWin, Power Term, or Apple Terminal, do need an operating system login ID. Changing this setting will define or remove the opearing system Login.
Home Directory This is a display only field just for information purposes.
Shell The Shell is the program run that provides the user with the ability to enter operating system commands. On Linux systems the /bin/bash Shell is most popular. You can use /sbin/nologin to prevent the user from logging in. Other special purpose shells are available.
Password The operating system password can be set or changed using this field. There is also an option to lock a user which prevents them from logging in. Locked users can be unlocked with this application.
Dynamo Tools Profile The Profile is the first program that is run when the user logs in. Typically it launches Dynamo. This setting indicates if the user is using the standard Dynamo Tools Profile program. There are options to replace the profile with the standard version, or to view/edit the user's profile.
.rhosts file This file is required when using user based authentication in conjunction with the pro5 or BBj pro5 data server. Normally the file is configured to provide unrestricted access, which means that all users may connect from any location. There are options to display/edit the .rhosts file, or change it to be unrestricted.
Alternate Profile Present

An alternate profile is actually an additional profile with the name .altsmsprofile which is run near the end of the standard profile. It can be used for selected operators or specialty devices such as a hand-held computer to run a menu system other than the standard Dynamo Tools menu. Note that this option is only used when there are selected users with special requirements. If all users require a special menu or initial program to run, then the /u/CDI/bin/profile.local file is used for those commands.

This option can be used to view/edit the alternate profile file, or to remove it.
