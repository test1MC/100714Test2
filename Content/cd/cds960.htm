Display System Status

 Last Revised: 03/02/16

This program displays information about all Dynamo and Dynaweb applications currently running.  It can also be run using the T option at any menu or Hotkey.  Several different sequences, or views, are available.  This application can also be used to terminate (kill) a hung or otherwise inaccessible application.  Supervisor level security is required to kill another process if not your own.  If the Double Vision remote control software is installed, then Display System Status can also be used by authorized operators to view and manage another operators session. Note that users who did not exit their application properly may still show an active session running here and may still be occupying a software license.

Display System Status Options
Terminal ID Lists all active, non-background, sessions in order of their Terminal ID.
IP Address Lists all active, non-background, sessions in IP Address sequence with the sessions that do not have an IP address appearing first.
Login ID Lists all active, non-background, sessions in Login ID sequence. This view is useful when looking for sessions of a particular operator.
Company + Menu Code + Program Lists all active, non-background, sessions in order of Company + Menu Code + Program sequence. This is useful when checking to see how many sessions are active for a specific company.
Job Start Date & Time Lists all non-background sessions in Job Start Date & Time. This is useful when trying to locate older sessions that may still be running and consuming a software license.
Elapsed CPU Time Lists all processes running, including non-Dynamo processes, listing the processes that are currently consuming the most CPU first. The operation will observe CPU usage for 1 minute before returning an analysis. If a system appears to be running more slowly than normal, this option can be useful for identifying a possible cause.
Dynaweb & Background Tasks Lists all background tasks including Dynaweb, BBj stored procedures, and BBj triggers.  Dynaweb tasks are typically very brief, often only a few seconds, unless they are creating some lengthy report.
Sessions currently with an error Lists all sessions that have recently encountered a program error and have not yet returned to the menu.  Useful for software support personnel to quickly identify a problematic session.
Purge session records for inactive processes Normally the entry for an active session is removed when the operator logs out of Dynamo.  However, there are times when the application aborts and does not log out properly.  In this case, the session record may still exist even though the application is no longer running.  Selecting this option will purge any session records for inactive processes.
Remote location & device status This option is only available when there are Virtual Private Networks (VPN's) established between the server location and branch offices.  This option will display VPN status information.  If any VPN's have been configured are inactive, then this option may take a short while to verify the VPN status.  The remote locations that appear on this option are maintained using 'Location & Device Maintenance' on the SMC menu.  See example below.

Highlighting an entry and touching the Enter key will display additional information about the session and offer additional options based on the security level and software installed.

 

The D=Double Vision option is used to view and optionally operate the selected session.  This requires authorization and is typically used by technical support personnel for training and problem resolution purposes.

The K=Kill this process option should only be used when the job cannot be aborted from the terminal running the job.  This option requires the operator to have "Supervisor" security status as specified in Operator Information Maintenance (SMS).  In order to kill a BBj process (Java-based version of Dynamo), you must first used the Basis Enterprise Manager to define an operator named 'killBBj' with a password of 'killBBj123'.  Check the permissions box named 'BBj Processes'.
