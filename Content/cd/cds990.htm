Documentation Manual Printing

 Last Revised: 02/11/06

This program is used to print system documentation for one or all systems for a certain company. Certain portions of the text may be included or excluded depending on the preference of the operator and the purpose for printing the documentation.  Note that most documentation is now kept online at http://www.excellware.com.  The information printed using Documentation Manual Printing is not kept current like the online documentation.

The following information is entered prior to printing the documentation. Defaults for each prompt have been set and are given below.

CDS990
System Code Enter the three character system code for which the documentation is desired. The default is 'SYS' which is the main system selector.
Company Name Enter the company name to print in the heading of each page of text.  The default is the name of the company from which the program is being executed.
Table of Contents Enter 'Y' to include, 'N' to exclude. The default is 'Y'.
Selectors Enter 'Y' to include the printing of the selectors as they appear on the terminal screen, 'N' to exclude. The default is 'Y'.
Detail Selectors Enter 'Y' to include the printing of the selectors with the detailed information as entered in Selector Maintenance. Since this program is primarily used to print documentation to be used by customers as a brief summation and reference as to what a program does, the default is 'N'.
Screen Formats Enter 'Y' to include the printing of the screen format that is displayed when the program is actually run. The default is 'Y'.
Documentation Text Enter 'Y' to include the printing of the text which provides a brief explanation of the programs purpose and operator instructions. The default is 'Y'.
Operator Instructions Enter 'Y' to include the printing of the general operating instructions for the operator. Topics covered include running programs from the menus, end system, backup procedures, etc. The default is 'Y'.
Missing Text/VDT List Enter 'Y' to include the printing of a list of all programs interfaced through the selector which do not have text and/or a VDT, when applicable. The default is 'N' since printing a missing text list is not the primary purpose of this program.
Subselectors Enter 'Y' to include the printing of any subselectors found when processing a main selector. As an example, if the system code being printed is SOP Sales Order Processing and this flag is 'Y', then all of the information which has printed for SOP (screen formats, text, etc.) will print for SOU Sales Order Processing Utility Functions, a subselector of the main selector, SOP. The default is 'Y'.
Program/File Names Enter 'Y' to include the printing of the six character program or file name next to the program title on the Table of Contents. The operator is only prompted for a response if the Table of Contents is being printed with this text. The default is 'N'.
Lower Case Option Enter 'Y' if your printer supports lower case printing. The default is 'Y'.
Blank Lines in Text Enter the number of blank lines to print between each line of text. The default is '0', which means the text will print single-spaced.

The setting of these flags depends on the purpose of printing the text. If the purpose is to fulfill a specific need of the operator, then most of the flags should be set to 'N'. As an example, if the operator needed to know which programs interfaced through the selector are missing text and/or a VDT, then all other flags should be set to 'N' to avoid printing unnecessary information and wasting paper. If the purpose is to provide a documentation manual for a new customer, then the flags should be set to the defaults as described above in order to keep the information about their system as concise and un-technical as possible.
