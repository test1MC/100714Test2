System Management Software

System Activity Log

 Last Revised: 02/11/06

The purpose of the system activity log is to give an account of who, when, and how long each program has been run since the last time the report was printed.  The System Activity Log is enabled using System Control Maintenance (SMC).  If you enable the System Activity Log, be sure to print and clear the data periodically, as it can consume quite a bit of disc space.

If this program is to be printed by an operator who does not have supervisor status, then a supervisor will be required to enter their operator ID and password after this report is selected from the menu.

The System Activity Log Screen

The operator has the option to print the report or to simply clear all system activity. The data should be cleared only if you do not require any of the reports. Once cleared, the report cannot be printed.

Optional Formats
Print/Clear Activity Log The operator has the option to print the report or to simply clear all system activity. The data should be cleared only if you do not require any of the reports. Once cleared, the report cannot be printed.
Detail by Terminal ID If requested, each program selected from the menu will be shown on the report in chronological sequence for each terminal. The date, operator ID, terminal ID, company code, program description, start time, stop time, and elapsed time will be printed.
Detail by Operator ID If requested, the same information listed above will be printed by operator ID, regardless of which terminal was used.
Summary by Terminal and Program If requested, every program run from each terminal will be summarized. For example, if the Sales Order Entry program was run for 15 minutes, 20 minutes, and 5 minutes, this report shows the total of 40 minutes for Sales Order Entry. The Terminal ID, program name and description, total elapsed time, and percentage of total system time is shown.
Summary by Operator ID and Program If requested, the same information listed above will be printed by operator ID, regardless of which terminal was used.
Summary by Program If requested, the same information listed above will be printed without any breakdown by terminal or operator. This option would tell you the combined time spent in each program by all operators on all terminals.
In addition to these optional formats listed above, the following sections are always printed.
Special events
This section of the system activity log shows any security oriented problems or programs not run to completion. This format includes the date, operator ID (if any), company code (if any), time, and comment. Some of the comments are described below.
ABNORMAL COMPLETION This means that the program shown did not finish properly. It may be due to some type of system failure or halt condition.
The following special events do not apply when using UNIX Operating Systems when there is a login at operating system level.
INVALID OPERATOR ID This indicates that a valid operator ID was not entered properly in the three tries given during the operator sign on procedure.
SECURITY VIOLATION This indicates that a valid operator ID was entered, but the password was not entered correctly after three attempts. The operator ID that was used is also printed.
SECURITY VIOLATION - SPV ID This indicates that after the operator ID was not entered properly after three tries, the supervisor ID was also not entered correctly after three tries.

Following any summary options selected, the total time used by terminal ID, operator ID, and company code is printed. This information should be useful in determining where workload is lightest and heaviest to aid in relocating or obtaining additional terminals.

If any of the two detail options were selected, the normal wide paper is required. The summary pages require only the narrow 8.5 inch wide paper.
