Location & Device Maintenance

 Last Revised: 01/04/15

This application is used to describe branch offices that are connected to the headquarters, over a Private Network, or a Virtual Private Network.  It is also used to describe devices that can be remotely managed including routers, PCs via VNC, printers via a web browser, servers via ssh using putty, etc.

The list of Locations is displayed using the 'Remote location & device status' option of Display System Status.

Location Code A six character identifier for each location
Location Name Descriptive Name that can by searched on using the Remote Location and Device Status option of Display System Status
Public Hostname or IP WAN hostname or IP used for connections to router
Router LAN IP Address Used to determine if the VPN tunnel is operational, and to connect to the router from the LAN or over VPN
VPN Active Y/N to indicate if there is a VPN normally connected to this location.  Used to verify the status and ping time of VPN's.

PC Requirements

 

The ssh and telnet connections use the free third party application called Putty. It is available from the downloads page of our web site. You should also download, extract, and open, the recommended Excellware Putty Default Settings which update the PC's windows registry with appropriate default values. You do not need to create a connection within Putty for each connection defined using Location & Device Maintenance. You may need to add the path to the putty.exe executable to the Windows program PATH.

The pptp connection options use the pptp client connection application that is included with Microsoft Windows. However, you do need to create one pptp VPN connection with a connection name DynamoTools. The hostname/IP Address, login, and password can be specified as default values, but they are not used when making a connection. Remember to uncheck the option to Use default gateway on remote network if you need access to the local area network while the pptp vpn is active.
