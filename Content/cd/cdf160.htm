Basis Data Dictionary Maintenance

 Last Revised: 01/10/14

This application can be used to:

  • Define files required by the Basis Data Dictionary,

  • Populate the Basis Data Dictionary for one or more files defined in the Dynamo Tools Data Dictionary,

  • Launch the character based Basis Data Dictionary maintenance program

The first option is used to create a folder for the Basis Data Dictionary files (if it does not exist), define the Basis Data Dictionary files (if they do not exist), and to launch the Basis character based Data Dictionary Maintenance program.  The Basis Data Dictionary files are defined in the XX/bbdict/ folder where XX is the two character Dynamo Company Code.  Click here to display the Basis documentation for the Basis Data Dictionary files.  Within the Basis Enterprise Manager, set the DATA variable to "/<localhost,port=1100>" and the DICTIONARY variable to "/<localhost,port=1100>/u/CDI/XX/bbdict/" when using the pro5 data server to access the data files and Basis Data Dictionary.  Note that /u/CDI should be replaced by the Dynamo Tools install directory (SMSDIR), and XX should be replaced by the two character Dynamo Company Code.

Note that it is recommended to use the Basis Enterprise Manager to view or change the Basis Data Dictionary, and not the character based version of the Basis Data Dictionary Maintenance.

The second option is used to replace the Basis Data Dictionary entry for any MKY, SKY or DIR data file described in the Dynamo Tools Data Dictionary.

It is recommended that you only create a Basis Data Dictionary for the files that you require.  Many third party applications, such as Microsoft Excel, Microsoft Word, Microsoft Excel, and WordPerfect retrieve the complete list of all entries in the Basis Data Dictionary.  This can cause a significant delay when there are many entries.

Effective 09/24/08, this application will populate the Basis Data Dictionary with the data file name as "(DATA)" plus the full path name to the data file.  This means that the data file must exist in order to Update the Basis Data Dictionary.

The file type specified in the Dynamo Data Dictionary must match the actual file type.

The field names defined in the Dynamo Data Dictionary are used to populate the Basis Data Dictionary.  The field names contained in the .tpl file are not used.

Although the Dynamo Tools Data Dictionary supports all file types, the fields that make up the keys are only described for MKY file types.

Effective 09/24/08, this application assumes the key to any SKY or DIR file is the first N characters of the record.  Although this is a popular convention, it is not always the case.  If the defined keysize is larger than the first N characters of the record, or there are field separators within the first N characters, or the key is not the first N characters, then you should use the Basis Enterprise Manager to describe the key structure.  Note that the Basis Data Dictionary requires the key be contained somewhere within the record.

Due to the limitations of SKY and DIR file types, the best practice is to use MKY file types for all data files.
