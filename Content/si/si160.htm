Serialized Inventory File Integrity Test

 Last Revised: 09/17/13

Excellware uses Serialized Inventory File Integrity Tests to find and correct any errors within an operator's serialized inventory files. If immediate repair cannot be made, the test reports and prints out errors that can be corrected later. These errors may relate to out-of-balance items, non-applicable characters appearing in certain fields, or incorrectly processed transactions. Typically, these tests are used after a customer's files have been converted during an upgrade or after a program revision. The operator does not need to run this application.
