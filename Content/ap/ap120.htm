A/P Recurring Invoice Generation

 Last Revised: 01/10/14

Selecting this program causes the invoices that were set up in Recurring Invoice Maintenance to be scanned, and those that need to be generated by the operator-specified date are added to the invoice data entry batch.   If the frequency of the recurring invoice is weekly, bi-weekly, or semi-monthly, multiple invoices may be generated during this process.

You will be given the option to select an existing data entry batch to which these recurring invoices will be added or to create a new batch.

Note: you must return to A/P Invoice Data Entry to modify or maintain a batch entry.

Generate Invoices Dated Through: The date you enter in this field depends on how often you generate invoices.  The date you enter should ensure that all invoices which need to be generated will be included. For example, if you typically generate invoices on the 1st of the month (e.g. 5/1), you should Generate Invoices Dated Through: 5/31.
