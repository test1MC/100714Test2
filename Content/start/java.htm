Java

 Last Revised: 12/28/13

Java is not required for applications running pro5 nor vpro5, however, Java is required when running BBj.  This document describes how to download, install, configure, and link a BBj installation to a particular Java version.

Java is a programming language developed by Sun Microsystems, IBM, and others.  It became an Oracle product when Oracle acquired Sun Microsystems. A compatible version of Java must be installed when using BBj.  Java is available in a run-time only environment called a JRE (Java Runtime Environment), and a full developers environment called a JDK (Java Developers Kit).

When installing Java on a server, always install the JDK and not just the JRE, as certain features of BBj, such as signing JAR files, require the JDK. Version 1.7.0_45 or later is recommended. For additional details, visit the Dynamo Tools Installation page.

If installing Java on a Microsoft Windows PC or Apple Mac for use with the BBj thin client, then the JRE can be installed.

Microsoft Windows

In some cases, multiple versions of Java must be installed to support applications other than BBj that are running on the same PC.  Oracle supports multiple major versions of Java, such as 1.5, 1.6, and 1.7, installed on a PC at the same time.  When multiple versions of Java are installed, the version used by web browsers is managed using the Java Runtime Environment Settings located on the Java tab of the Java Control Panel. (The Java Control Panel can be opened in both Windows 7 and 8.1 by searching for "configure Java" and clicking on the search result.)
