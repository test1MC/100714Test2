Dynamo Initial Setup

 Last Revised 06/30/16

Dynamo has the powerful ability to link many business processes to each other so that your business runs as a whole. As a few examples, Dynamo links purchase orders to inventory, inventory to sales orders, sales orders to customers and payments, and all of this to the general ledger. On top of that, the Dynamo experience is customizable. As such, there is a need to prime Dynamo with as much information about your company as possible in order to make all business components work together. The following set of procedures is meant to do just that.

If this is your first time using a character-based application, you may want to view Excellware's brief tutorials covering general navigation and data entry. These include data entry conventions, navigating menus, entering dates, selecting records from a list, and selecting records from a grid.

Note that each step below should be followed in order. For more detailed help, there is a link in each step to the Dynamo documentation page for that program.

 

Menu Code Program Brief Description of Step

SMC

 Company Information Maintenance Enter essential company information such as company name, address, phone/fax numbers, etc.
CWP

 

 Dynamo Parameters Maintenance
  1. Under Period Parameters, define the fiscal year, current month, and current year.
  2. Under Dynamo Options, enable or disable a series of attributes related to inventory.
  3. Under Dynamo Settings, set additional options related to inventory.
IML Product Category Maintenance Product category names should be generalized and brief. Examples might be Sales, Rentals, Parts, Service, etc.
IML Product Class Maintenance Product classes are more specialized subsets of product classes. For example, under a "Sales" product category, product classes could be created for the types of products sold such as tools, apparel, fasteners, etc.
GLU G/L Parameter Maintenance Define the structure of general ledger account numbering and, if applicable, whether the account numbers will refer to multiple company locations for easy distinction during reporting.
GLU G/L Account Group Maintenance Define ranges for general ledger account groups. General ledger account numbers—defined in a later step—that fall in these ranges will inherit the same broad accounting category as the range defined here (e.g. Assets, Liabilities, Equity, etc.). You may wish to print a hard copy of this screen (F5) for reference when creating actual G/L accounts in the next step.
G/L G/L Account Maintenance This is where the company chart of accounts is created. Define the specific account numbers that will be associated with each general ledger account such cost of goods, sales revenue, utility expenses, etc. Be sure that the account number falls in the appropriate range for the accounting category type (e.g. shipping expenses should be numbered in the "Expense" group as defined in the previous step).
GLU G/L Period Ending Date Maintenance If needed, change the closing dates for each general ledger posting period. Periods can also be opened or closed to allow or prevent further transactions being posted to the period.
GLU G/L Beginning Balance Maintenance

If beginning to use Dynamo in the middle of a fiscal year, some general ledger accounts presumably have balances that should be provided here. Specify the accounts and corresponding balances so that Dynamo can produce accurate financial data.
VML Vendor Maintenance Dynamo needs a vendor account for the appropriate sales tax authority. A "Miscellaneous" vendor can also be created for non-routine transactions and those that do not require a vendor to be on file. Other vendors can be set up later as needed.
SOU SOP Parameter Maintenance Set the default fields and values that Dynamo will use for new sales orders.
CML Customer Maintenance Create customer accounts. A "Miscellaneous" or "Quick Sale" customer account should be created for customers who do not need to be (or wish to be) on file. Other customer accounts can be set up later as needed.
CWP Branch Information Maintenance If applicable, enter information about individual company branch locations.
APU Bank Code Maintenance Enter information for any bank account(s) that Dynamo will interact with.
ARU Cash Drawer Maintenance Enter information related to any cash drawers used at points of sale including beginning balances and branch locations.
CWP Workstation Information Maintenance Set default values for any workstation running Dynamo such as default branch location, cash drawers used, printers available, etc.
CWP Warehouse Information Maintenance If applicable, enter information for each company warehouse.
GLU G/L Interface Maintenance Define how individual accounts in the chart of accounts will flow into the general ledger.
SMC Operator Information Maintenance Create and edit Dynamo users. Also assign security levels, passwords, and other user-specific attributes.
SOU Commission Code Maintenance If employing a commission-based sales force, create commission parameters that can be assigned to salespersons.
CML Salesman Maintenance Create entries for individual salespersons and assign commission codes if applicable.
SOU Sales Tax Control Maintenance Enter information needed for Dynamo to automatically calculate sales tax on sales orders.
VML Buyer Maintenance Create entries for buyers within the company. These buyers can then be assigned to vendors and authorized to make purchase orders to that vendor on the company's behalf.
ARU Payment Method Maintenance Create and maintain payment methods that are available during the processing of a sales order.
