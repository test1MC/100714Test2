Dynamo

Supported Workstations

 Last Revised 08/24/15

Dynamo can be run from a variety of workstations including personal computers running Microsoft Windows, Axel thin client workstations, Apple Mac computers, and legacy Wyse and IBM terminals.

Personal Computers
  • Microsoft Windows XP Professional, XP Home, Vista, Windows 7, or newer.
  • Dynastart - Launch Dynamo from web browser.  No terminal emulator software needed on PC.  Requires free Java download from Oracle/Sun.
  • Recommended terminal emulator, supports all features: FacetWin
  • Alternate terminal emulator, supports limited features: putty
Axel thin client workstations
  • Supports both character mode and graphical (GUI) mode
  • Character mode requires no additional software
  • GUI mode - Dynastart - Launch Dynamo from web browser.  No terminal emulator software needed.
  • GUI mode - limited features using Linux xterm program
  • GUI mode - all features using PowerTerm Interconnect for Linux
Apple Mac computers
  • GUI mode - Dynastart - Launch Dynamo from web browser.  No terminal emulator software needed.
  • Character mode - limited features using Apple Terminal program
  • Character mode - supports all features using PowerTerm Interconnect for Mac OS-X
Apple iOS devices
  • Apple iPhone or iPad
  • Recommended terminal emulator, supports limited features: Century Software TinyTERM ITX2 for iPad
Serial Terminals (supported, but not recommended)
  • Wyse 60 and 160 terminals
  • IBM 3151 and 3153 terminals

Dynastart
concept Dynastart is the newest (and perhaps easiest) way to launch Dynamo.  This is because the only software needed is Java, which is typically already installed on PCs and Apple Macs.  No terminal emulation software such as FacetWin, PowerTerm, or Apple's terminal program is required.
requirements
  • Java runtime environment (jre) version 1.7.0_51 or higher installed on workstation.  On a PC you can check (and upgrade) the version of Java installed using the Windows Control Panel, Add/Remove Programs option. Click here for the Java download page. For Windows PC's download the link that reads: jre-7u51-windows-x64.exe.
  • BBj version 13.13 or higher installed on the server.  You can check the version of BBj installed using the I (Information) option at any Dynamo menu.  Contact Excellware to upgrade BBj.
  • Dynamo Tools - Developer Components dated 02/15/14 or later installed on the server.  You can check the version of the System Management Software installed using the V (Versions) option at any Dynamo menu.
  • Dynastart configured on server.  Excellware will do this.
  • Each operator name and password must be defined.  Excellware will do this, but end operators can change their password at any time after logging in.
Installation Open a browser such as Chrome, Firefox, or Internet Explorer and enter the address provided by Excellware, i.e.., http://dynamo.excellware.com:8888/jnlp/dynamo.jnlp
 

You may get a small box as shown, that requires you to click the Keep button.

This example is when using the Chrome browser. You can close the browser once this small file in downloaded.

Please see below for how to download using the newest version of Safari.
 

Over the next few minutes, the programs will be downloaded, verified, and installed on your workstation.
 

When the download and verification process completes, check the 'Do not show' box, then click Run.
 

Click the Yes button to Install the Certificate Authority.
 

Check the 'I accept the risk' box, then click Run.
 

Dynamo is now installed and will launch. A shortcut will be installed on your desktop. You can click that icon to launch Dynamo after the current session is closed.

Using PowerTerm on an Apple Mac, you can also print from Dynamo to a printer attached to the Mac.  This is sometimes referred to as slave printing.  You can also print to a printer attached to the same subnet as the Mac.  For example, the printer could be wireless or connected to a router or another workstation.  Most laser and inkjet printers are supported.  This configuration is often used when printing from home.

Configuring Dynamo printing using PowerTerm on an Apple Mac
1 Install and configure the printer and confirm that the Mac can print to it using any Mac application
2 Start PowerTerm on the Mac, and click Terminal Setup, then the Printer Tab.  Verify that all setting match those shown in the screen shot below.
3 Click the Advanced Printing button, and confirm that all settings match those shown in the screen shot below.
4 Close the Advanced Printing and Terminal Setup windows.
5 Click File, then Print from the PowerTerm menu.  You should be able to select the desired printer from the printer drop down box.
6 Test printing from Dynamo.  The printer is typically described as Slave or Local printer.  See the screen shot below for an example of adding a Dynamo Printer using Printer Control Maintenance (PFM) or contact Excellware for assistance adding your particular printer to the Dynamo printer list. 
7 Save the PowerTerm configuration settings.

Configuring a Dynamo Local Printer, i.e., printer attached to workstation
Printer ID Any 4 character Printer ID that starts with a letter can be used.   Note that the printers are displayed in the selection list in Printer ID sequence.
Description "Local Printer" or "Slave Printer" or "Mike's Printer", or other appropriate description.  Note that multiple operators can share a common Local Printer, as it is local to the Workstation being used
Type H2100 is often a good choice for a black laser printer that prints on one side of the paper (simplex).  H210D is the printer type that support two sided printing (duplex)
Print Queue Not used for local/slave printing
Connection Not used for local/slave printing
Device Always use /dev/tty for local/slave printing
Status Set to E to Enable this printer, i.e., allow operators to select and print to if.  Set to D to Disable the printer which might be needed if the printer is temporarily not available.  Note that you can use Printer Preference Maintenance (PFM) to limit the printers that appear on the list, and to specify the default printer.
Default Method Set to D 'Direct to Device' for local/slave printing
Timeout Seconds Can often be left at zero.  If you encounter printer busy messages including error zero's, then increase this parameter.
